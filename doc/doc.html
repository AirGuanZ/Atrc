Atrc Renderer Documentation

Building

Dependencies

编译Atrc时须额外准备的依赖项如下：

  • cmake (>=3.10)
  • 支持C++17标准主要特性的MSVC或clang++（g++理论上可用，但未经测试）

已经由项目文件自带的依赖项如下：

  • agz-utils，提供数学计算、图像加载等基本功能
  • cxxopts，用于命令行参数解析
  • nlohmann json，用于JSON配置文件的解析
  • stl reader，用于解析STL模型文件
  • tiny obj loader，用于解析OBJ模型文件

可选的、在构建时由项目自动下载的依赖项如下：

  • Embree 3.5.2，用于加速射线与几何体的求交

可选的、需额外准备的依赖项如下：

  • oidn，基于机器学习的降噪滤波器

CMake Options

注意到OIDN只支持64位程序，因此若启用了OIDN库，必须以64位模式构建程序。

Example

以Windows环境为例，若启用Embree、OIDN等可选功能，典型的构建过程如下：

  1. 在PowerShell中运行命令：

    1
    git clone --recursive --depth=1 https://github.com/AirGuanZ/Atrc

  2. 编译OIDN或在此处下载预先编译好的版本，设包含其CMake Config的文件路径为X

  3. 在PowerShell中继续运行命令：

    xxxxxxxxxx
    3
    1
    mkdir build
    2
    cd build
    3
    cmake -DUSE_EMBREE=ON -DUSE_OIDN=ON -DOIDN_PATH="X" -G "Visual Studio 15 2017 Win64" ..

这会在Atrc/build下生成Visual Studio 2017的解决方案文件。第一次编译时会从网络上Embree库，此时应保持网络良好。

Usage

Components

编译Atrc得到的结果由以下几部分构成：

  1. CLI，为渲染器的命令行可执行程序
  2. obj_to_scene，用于将一个带材质的obj文件转换为Atrc可用的JSON描述格式
  3. 一个或多个以共享库形式出现的材质插件

本节主要介绍CLI的使用，即其命令参数的含义和场景配置文件的编写。

Rendering Settings

本节描述渲染配置文件中的rendering一项包含哪些字段。

Scene

本节描述类型为Scene的字段有哪些可取的类型值。

default

注意到env字段的类型也是实体Entity，这是由于在Atrc中光源也是一种特殊的实体。

EntityAggregate

对于射线和实体间的求交，Atrc使用两级加速数据结构（参考RTX的设计），其中实体内部为一级，实体间为一级。EntityAggregate就是实体间加速数据结构对应的字段类型，本节描述它有哪些可取的类型值。

native

不包含任何字段。表示暴力遍历所有实体。

bvh

用BVH树将实体组织起来。

Camera

本节描述类型为Camera的字段有哪些可取的类型值。

pinhole

小孔摄像机的渲染结果是上下颠倒的，因此通常会和类型值为“flip”的后处理器配合。

Entity

本节描述类型为Entity的字段有哪些可取的类型值。

diffuse

几何漫射光源，即具有几何形状的光源，表面上每一点向所有方向发射相同的辐射亮度。

geometric

普通的物体，可指定其几何形状、材质和内外介质。

dir

环境光的一种，在指定的锥形立体角范围内的方向上有值为常量的辐射亮度。

我们知道，环境光可以由方向的函数描述，表示沿方向向量的环境光辐射亮度。设dir字段值为，radiance字段值为，range字段取值为，则dir环境光对应的函数为：

env

环境光的一种，由纹理给出它在每个方向上的辐射亮度。

native_sky

环境光的一种，表示自上而下颜色渐变的天空。

aggregate

环境光的一种，表示将多个其他类型的环境光叠加。

FilmFilter

渲染图像的过程也是对图像平面进行采样-重建的过程，FilmFilter就是用于重建的滤波函数，详细可参见此文。若对这部分内容不甚了解，推荐使用最简单的重建滤波函数：

xxxxxxxxxx
2
1
"type": "box",
2
"radius": 0.5

box

最简单的正方形滤波函数，半径取为0.5时恰好和单个像素重合。

gaussian

Gaussian滤波函数。

Film

用于存储渲染得到的图像。

native

最基础的的点阵存储，支持位置、深度、材质颜色和法线等属性构成的G-Buffer。

filtered

支持任意重建滤波函数的点阵存储，支持位置、深度、材质颜色和法线等属性构成的G-Buffer。

Fresnel

表示物体表面材质中的fresnel项。

always_one

值永远为1的fresnel项，表示能量的反射/折射比为。

dielectric

电介质的fresnel项。

conductor

导体的fresnel项

Geometry

用于描述物体的几何形状。

绝大部分几何形状都包含类型为[Transform]的transform字段，即一系列Transform的序列。值得注意的是，序列中后面的Transform先作用到物体上，前面的Transform后作用到物体上。

disk

单面圆盘。

double_sided

双面适配器，它将一个单面几何形状转为双面，主要适用于圆盘、三角形等面片形式的几何形状。

quad

单面四边形，由两个三角形和构成。

sphere

单面球体，面向外侧。

triangle

单面三角形。

triangle_bvh

以BVH树组织而成的三角形网格形状，在启用了Embree时使用Embree实现，否则使用自行构建的BVH树。

triangle_bvh_embree

使用Embree实现的三角形网格形状，参数和triangle_bvh相同。

triangle_bvh_noembree

不使用Embree实现的三角形网格形状，参数和triangle_bvh相同。

Material

材质描述了物体表面与光的交互过程，本节介绍类型为Material的字段可取的类型值。

disney_reflection

Disney Principled BRDF的完整实现，有的参数含义我不知道怎么翻译比较合适，所以最好参考原文。

disney

不带BSSRDF的Disney Principled BSDF，具体可参考原文。

frosted_glass

磨砂玻璃材质，详情请参考Microfacet Models for Refraction through Rough Surfaces。

glass

光滑玻璃材质。

ideal_black

完全吸收材质，也就是一团黑。

ideal_diffuse

理想漫反射，反射到各方向的辐射亮度均相同。

mirror

理想镜面。

mtl

由Wavefront MTL文件描述的材质，也就是标准的漫反射+高光。

add

把多个材质直接叠加到一起。上图中的第三幅是使用add和scale配合，将前两幅图像中的物体材质按的权重求和得到的效果。

scaler

线性地缩放一个材质的反射/折射亮度。

mirror_varnish

表面绝对光滑的清漆，其内部必须是不透明材质。

lib

以外部共享库形式存在的材质，详情请见后文。

Medium

物体内部介质的描述。

void

真空，也就是什么介质也没有。

absorbtion

无散射的介质。

homogeneous

各处散射性质均相同的介质。

Post Processor

在渲染完成后用于图像处理的后处理器。

flip

翻转图像。

gamma

对图像进行gamma校正。

gamma和inv_gamma只需给出其中一个即可。

oidn_denoiser

使用OIDN对图像进行降噪，注意最好使用支持G-Buffer的Flim和Renderer。

save_gbuffer_to_png

将G-Buffer保存至png文件。

当使用的Film或Renderer不支持G-Buffer时，保存的结果是无意义的。

save_to_png

将图像保存至png文件。

gamma和inv_gamma只需给出其中一个即可，若都未指定，则不进行gamma校正。

Renderer

渲染算法。

pt

最传统的路径追踪，可以通过integrator来指定使用怎样的追踪策略。

当工作线程数时，设硬件线程数为，则将使用个工作线程。比如可以将worker_count设置为-2，表示留两个硬件线程，把其他硬件线程都用起来。

light

Adjoint Particle Tracer的实现，收敛很慢，主要为学习用途。

在追踪一条路径时，有三个追踪参数min_depth，max_depth和cont_prob值得注意。当追踪深度超过min_depth后，将以cont_prob为通过概率使用Russian Roulette策略来终止路径；而在路径深度超过max_depth时，它将被无条件终止。

PathTracingIntegrator

PathTracingIntegrator用于描述path tracing算法中追踪何种路径。

direct

使用Multiple Importance Sampling技术计算直接光照。

native

没有任何优化技巧的暴力路径追踪。

isolated

用于突出展示个别物体的追踪策略，可单独指定背景纹理。

mis

使用Multiple Importance Sampling技术优化的路径追踪。

native_vol

没有任何优化技巧的暴力路径追踪，支持介质渲染。

mis_vol

使用Multiple Importance Sampling技术优化的路径追踪，支持介质渲染。

ProgressReporter

用于输出渲染进度。

stdout

输出到标准输出流。

Sampler

随机数采样器。

native

最基本的随机数采样器，所有样本间都是独立的。

Texture

纹理对象。

所有纹理都包含以下字段（这些字段就不在后面每种纹理中列出了）：

值得注意的是，inv_v, inv_u, swap_uv和transform都是对uv的变换，其中transform最先起作用，随后swap_uv起作用，inv_u和inv_v最后起作用。而transform序列中，也是前面的变换后起作用，后面的变换先起作用。

checker_board

棋盘网格纹理，主要用于测试uv。

grid_count和grid_size中只需给出一个即可，在语义上它们满足grid_count * grid_size = 1。

constant

常值纹理，即在它的任何部位采样都会获得相同值。

hdr

从.hdr文件中加载出的纹理。

image

从各种常见图像文件格式（.bmp，.jpg，.png，.tga等）中加载出的纹理。

scale

对另一纹理对象进行线性放缩的wrapper。

Transform

对三维坐标的仿射变换。

translate

rotate

rad和deg中只能有一个。

rotate_x

绕轴旋转。

rad和deg中只能有一个。

rotate_y

绕轴旋转。

rad和deg中只能有一个。

rotate_z

绕轴旋转。

rad和deg中只能有一个。

scale

各向等比例缩放。

Transform2

对二维坐标的仿射变换。

translate

rotate

rad和deg中只能有一个。

scale

各向等比例缩放。
